01　疑似言語とは

1. この授業の目標

この授業では、基本情報技術者試験（科目B）のアルゴリズム分野に対応できる力を身につけることを目標とします。 科目Bでは「疑似言語」と呼ばれる特殊な言語でプログラムが書かれており、それを正しく読み解く力が問われます。

科目Bの出題構成（参考）
全16問中、アルゴリズム問題が14問・情報セキュリティが2問。 アルゴリズム問題はすべて疑似言語で出題されます。

2. 疑似言語とは

疑似言語とは、IPA（情報処理推進機構）が基本情報技術者試験のために独自に設計した、仮想のプログラミング言語です。

疑似言語の特徴

項目内容
設計者IPA（情報処理推進機構）
目的試験専用。アルゴリズム理解力を公平に評価するため
実行環境存在しない（動かして確認できない）
規格JIS・ISOには準拠していない独自仕様
文法の特徴日本語を使った比較表現、← による代入など
重要：疑似言語は暗記より読解力が問われます。 処理の流れを頭の中でトレース（追いかける）する練習が最重要です。

3. 主な構文一覧

● 変数の宣言

整数型: num
実数型: rate
文字列型: name
論理型: flag

● 代入（← を使う）

num ← 10
name ← "Taro"
flag ← true

● 条件分岐（比較は日本語で表現）

if ( num が 5 より大きい )
    name ← "OK"
endif
if ( num が 5 より大きい )
    name ← "OK"
else
    name ← "NG"
endif

● 繰り返し（while文）

while ( num が 10 未満 )
    num ← num + 1
endwhile

● 繰り返し（for文）

for ( i を 1 から 5 まで 1 ずつ増やす )
    sum ← sum + i
endfor

● 関数の定義と呼び出し

○整数型: add(整数型: a, 整数型: b)
    整数型: result
    result ← a + b
    return result

呼び出し例：

total ← add(3, 5)   // total に 8 が入る

4. 理解度チェック問題

問題01-1：疑似言語の定義

疑似言語について正しく説明しているものを1つ選びなさい。

解説を表示 正解：ウ
疑似言語はIPAが試験用に設計した仮想言語で、実行環境は存在しません。 JavaやPythonなどを参考に設計されていますが、いずれの言語とも異なります。

問題01-2：疑似言語の用途

疑似言語が使われる理由として最も適切なものを選びなさい。

解説を表示 正解：イ
疑似言語は特定の言語の実装スキルではなく、アルゴリズムの読解力・論理的思考力を測るために設計されています。

問題01-3：疑似言語の特性（正しくないものを選ぶ）

疑似言語に関する記述として正しくないものを1つ選びなさい。

解説を表示 正解：ウ
疑似言語はWeb開発などの実務で使われるものではなく、試験専用の仮想言語です。

問題01-4：構文の知識

疑似言語で条件分岐を表現する正しい構文はどれか。

解説を表示 正解：イ
疑似言語では比較演算を日本語で表現し（「が 5 より大きい」など）、代入には ← を使います。 ブロックの終わりには endif が必要です。
