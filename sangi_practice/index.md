## 学習の進め方（おすすめ）

1. **番号順に解く**: LV1 → LV5 の順に並んでいます。やさしい問題で「入力 → 計算 → 出力」の型に慣れてから、少しずつ難しい問題へ進みましょう。
2. **まず自分で考える**: いきなり答えを見ず、5〜10 分は自分で考えてみる。手が止まったら、各問の「学習アドバイス」を読んでヒントにしましょう。
3. **例で確かめる**: 提出する前に、問題ページの入出力例で自分のプログラムを動かし、答えが合うか確認するクセをつけましょう。
4. **1 言語で解けたら別言語でも**: 同じ問題を C でも Java でも書いてみると、言語の違いと「考え方は同じ」ことの両方が見えてきます。
5. **LV3 以降が本番**: GCD・BFS・DP などは基本情報やハッカソンの頻出テーマ。最初は難しくても、ここを越えると一気に力がつきます。じっくり取り組む価値があります。
