# 講義資料サイト プロジェクト概要

このリポジトリは、静岡理工科大学の青木伴暢先生による講義資料サイトのソース一式です。
HTMLファイルが各講義1ページに相当し、Cloudflare Pages で公開されています。

## 公開URL

https://tomoton-lecture-materials.pages.dev/

## ホスティング構成

| 項目 | 値 |
| --- | --- |
| GitHub | `tomotonton/tomoton-lecture-materials` |
| ホスティング | Cloudflare Pages |
| Production branch | `main` |
| 自動デプロイ | main へ push すると Cloudflare が自動ビルド |
| 認証情報 | `OneDrive/_aoki-lecture-materialsの資料/環境まとめ.txt` に記載 |

> ⚠️ `環境まとめ.txt` 末尾に GitHub Personal Access Token が記載されています。
> 公開リポジトリにコミットしないよう注意してください。

## ディレクトリ構成

```
C言語/              … C言語の講義HTML
C++(CPP)/           … C++の講義HTML
Java/               … Javaの講義HTML
アルゴリズム/        … アルゴリズム講義HTML（quiz-firebase.js もここ）
C言語_番外編/        … 番外編コンテンツ
tools/              … ビルドツール
  generate-index.mjs … 各フォルダの index.html を自動生成
index.html          … サイトトップ（自動生成）
package.json
```

## 公開フロー

```
1. HTMLファイル作成（拡張子は必ず .html）
        ↓
2. node tools/generate-index.mjs を実行（index 再生成）
        ↓
3. SourceTree で確認 → commit
        ↓
4. main ブランチへ push
        ↓
5. Cloudflare Pages が自動デプロイ
        ↓
6. シークレットウィンドウ or ?v=YYYYMMDD でキャッシュ回避して確認
```

### よくあるトラブル

- **C++フォルダ等が空に見える** → `.md` のまま置いている／index 再生成し忘れ
- **push したのに変わらない** → Cloudflare が最新コミットをデプロイできていない／ブラウザキャッシュ
- **再実行したのに古いまま** → 再実行は同じコミットを再ビルドするだけ。新コミットを push し直す

## Firebase 連携（クイズ・ランキング）

| 項目 | 値 |
| --- | --- |
| プロジェクト名 | `aoki-lecture-materials-quiz` |
| データベース | Realtime Database (asia-southeast1) |
| 共通ライブラリ | `アルゴリズム/quiz-firebase.js` |

### 既存のデータパス

| パス | 用途 |
| --- | --- |
| `answers/{qid}/{bucket}/{choice}` | クイズの選択肢別カウント（時間バケット集計）|
| `meta/{qid}/resetAt` | クイズのリセット時刻（管理者操作）|
| `bhScores/{dir}_hard/{key}` | 2進16進トレーニングの歴代ランキング（`dir`=b2h/h2b、値=`{handle,score,ts,uid,comment?}`）|
| `practiceStats/{id}/attempted/{uid}` ・ `.../passed/{uid}` | C/C++練習問題の挑戦・合格人数（ユニークなuid数で集計、`id`=問題id）|

### クイズの仕組み

- 各HTML内の `.quiz-choices[data-qid]` が `quiz-firebase.js` によって初期化される
- 学生の回答は localStorage と Firebase の両方に保存
- 過去48時間の集計を、**選択肢ブロックの右側の専用領域に横棒グラフ**（選択肢ごと・得票数・%・色分け：正解=緑/自分の誤答=赤/その他=青）でリアルタイム表示。回答後に右領域が出現し、狭い画面では選択肢の下に回り込む（自前描画・Chart.js不使用。選択肢ボタン自体は従来通り）
- 管理者は右下の「データリセット」ボタン（キーワード `ignas`）でクリア可能

### Firebase セキュリティルール（現状・2026-06 時点）

Realtime Database のルールは **全パス公開（read/write 全許可）** で運用中（教室内利用のため許容）。Firebase コンソール → Realtime Database → ルール の内容は次の通り：

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

> この1つで `answers`/`meta`（クイズ）・`bhScores`（2進16進）・`practiceStats`（練習問題）すべてが読み書き可能。
> **新しいデータパスを増やしてもルール変更は不要**（全許可のため）。
> ⚠ 公開ルールなので誰でも読み書きできる。不正投稿のリスクはあるが教室用途で許容。管理リセットは各画面の右下⚙（キーワード `ignas`）または Firebase コンソールから。

# 個別仕様

## 2進数・16進数変換トレーニング

### ファイル

`C言語/03-おまけ_2進-16進変換トレーニング.html`

### 背景

- 30秒・0〜F・両方向ランダム のみの初期版は難しすぎた
- 学生（A26043 望月啓希さん）から「制限時間が短く考えてる間にゲームオーバー・暗記ゲーになる」とフィードバック
- 範囲を狭める／時間を伸ばす／方向を片方向に絞ることでステップアップ式に再設計

### 構成

#### トップレベル: 出題方向（2つの独立ゲーム）

| ID | 方向 | 答え |
| --- | --- | --- |
| `b2h` | 2進数 → 16進数 | 16進1桁 |
| `h2b` | 16進数 → 2進数 | 2進4桁 |

#### 各方向内のレベル（4段階）

| ID | レベル | 時間 | 範囲 | 備考 |
| --- | --- | --- | --- | --- |
| `practice` | 📚 練習 | なし | 0〜7 | 変換表ボタンあり |
| `lower` | 🌱 しただけ | 60秒 | 0〜7 | 2進数の先頭ビットが0の組 |
| `upper` | 🌿 うえだけ | 60秒 | 8〜F | 2進数の先頭ビットが1の組（A〜F含む） |
| `hard` | 🔥 チャレンジ | 60秒 | 0〜F | **ランキング対象**（全16種類） |

> 「うえだけ」を分けたのは、A〜Fの文字対応＋2進数で `1xxx` の処理という二重の難しさが集中する苦手ポイントだから。

### 主要機能

- **スマートキーパッド**: 2進答えなら `0` `1` `←` の3キー、16進答えなら `0〜F` `←` の17キー
- **自動判定**: 必要桁数（2進=4桁／16進=1桁）入力で即判定。「判定」ボタン押下不要
- **ハンドルネーム入力**: ランキング登録に使用（最大12文字、空欄ならランキング非登録）
- **過去24時間履歴**: レベル別に正解数・回答数・正答率を localStorage で記録

### ランキング仕様

- **対象は チャレンジレベル のみ**（練習・しただけ・うえだけはランキングなし）
- **左右2パネル並列表示**：
  - 左: 2進数 → 16進数 のランキング
  - 右: 16進数 → 2進数 のランキング
  - 両方向を**同時に**購読・表示 → 両方の自分の位置が一目で分かる
  - 現在プレイ中の方向のパネルは青枠でハイライト
- **得点分布ヒストグラム** で表示（順位表ではなく分布グラフ）
  - X軸: スコア（0〜最高点）
  - Y軸: その点数の人数（バーの高さ）
  - 自分の自己ベスト位置は**橙色＋▲マーカー**で強調
- **歴代TOP10ランキング表**（順位・ハンドル・点・ひとことコメント）も併記
- パネル上部に「あなたのベスト N 問 / プレイ M 回 / 全 K 人中 X 位」を表示
- **歴代（全期間）** の記録を集計（記録は消えずにずっと残る）
- 同じユーザー（同 uid）の重複は最高スコアのみ採用
- 順位は同点同順位（より高い点数の人数 + 1）
- モバイル（max-width: 520px）では2パネル縦積み

### 上位ランクイン演出（お祝い・ひとこと）

- チャレンジ終了時、**自己ベスト更新 かつ 歴代TOP10入り** で全画面お祝い演出（canvas花火＋祝福カード）
- 歴代1位（他全員を上回る新記録）は「🎆 世界新記録 🎆」、それ以外は「🎉 歴代 N 位 ランクイン！」
- 演出内で**ひとことコメント**（任意・最大30文字）を入力でき、`update()` でそのレコードに追記 → ランキング表に反映
- 発火判定は `computeRank(mode, newScore)`（送信前の `lbCache` から各uidベストを集計し `rank`/`isImprovement`/`isWorldRecord` を算出）
- 花火は外部ライブラリ不使用の自前canvasパーティクル（`startFireworks`）

### Firebase 購読パターン

- `currentLevel === "hard"` に切り替わったとき、`bhScores/b2h_hard` と `bhScores/h2b_hard` の両方を購読
- 方向タブ切り替え（`currentMode` 変更）では**再購読しない**。アクティブパネルのハイライトと「歴代自己ベスト」カードの値だけ更新
- 非チャレンジレベルへ切り替えるとき購読解除

### Firebase データ構造

```
bhScores/
  b2h_hard/{pushKey}: { handle, score, ts, uid, comment? }   ← 2進→16進 チャレンジ
  h2b_hard/{pushKey}: { handle, score, ts, uid, comment? }   ← 16進→2進 チャレンジ
```

| フィールド | 内容 |
| --- | --- |
| `handle` | ハンドルネーム（最大12文字、trim済み）|
| `score` | 正解数（整数）|
| `ts` | 投稿時刻（ミリ秒タイムスタンプ）|
| `uid` | 匿名ユーザーID（localStorage、自分の記録識別用）|
| `comment` | ひとことコメント（任意・最大30文字。上位ランクイン演出で入力時のみ付与）|

### localStorage キー

| キー | 内容 |
| --- | --- |
| `bhUserId` | 匿名ユーザーID（初回生成、永続）|
| `bhHandle` | ハンドルネーム |
| `bhHist_{level}` | 過去24時間の回答履歴（レベルごと、`{ t, ok }[]`）|

> ランキング系は Firebase が正本。localStorage は使わない。
> 個人ベスト表示も Firebase の自分のレコードから算出。

### 設計判断の経緯

1. 最初は30秒・0〜F・両方向ランダムのみ → 難しすぎた
2. **練習・かんたん・ふつう・チャレンジ** の4レベル制を導入（範囲と時間を段階化）
3. 「両方向ミックス」も検討したが、**方向判別のワンクッション**が「考えてる間に時間切れ」の主因と判断し、片方向のみに
4. 当初は過去10日窓だったが「歴代を残してさらに上を目指したい」との要望で**全期間（歴代）保存**に変更。上位ランクインは花火＋ひとことコメントで祝う
5. 練習・かんたん・ふつうは個人練習用、競争はチャレンジに集約 → ランキングは hard レベルのみ

### Firebase セキュリティルール

`bhScores` パスは read/write 許可が必要。**2026-06 時点で DB ルールは全パス公開に設定済み**のため追加設定は不要（→「Firebase 連携」内の「Firebase セキュリティルール（現状）」を参照）。

不正なスコア投稿のリスクはありますが、教室内の用途なので許容範囲。
管理者によるデータリセットは Firebase コンソール、または各画面の右下⚙（キーワード `ignas`）から実行する想定。

## C言語資料の方針（純粋なC / C89）

- **変数宣言は必ず関数の先頭でまとめて行う**（C89スタイル）。`for (int i …)` のようなループ内宣言や、実行文（printf/scanf/代入など）の後での宣言はしない。ループ変数は `int i;` を先頭で宣言してから `for (i = 0; …)` とする。
- 変数の単元（`02_C言語の基礎_変数とデータ型.html` の 02-3）に、このルールの説明（良い例／悪い例）を記載済み。
- 改行は資料の慣習で **円記号 `¥n`（U+00A5）** で表記（バックスラッシュ `\n` ではない）。日本語Windows向けの既存仕様なので、勝手に `\n` へ置換しないこと。
- 新規作成・改修するC言語のコード例も、上記に必ず合わせること。
