04_enum（列挙型）試験範囲

対象：基本データ型を一通り学んだ後 ／ 「決まった選択肢の中から1つを選ぶ」型を理解します

■ 1. enum とは

enum（列挙型）は 「決まったいくつかの値のうちのどれか」 を表す型です。曜日、信号の色、トランプのスート、ゲームの状態など、取りうる値が限定的なものを表現するのに使います。

int で代用した場合（古い書き方） 
// 曜日を int で表現
int MON = 1;
int TUE = 2;
// ...
int today = MON;

// 問題点：
//  - int に何でも代入できてしまう
//    today = 999; // バグだがエラーにならない
//  - 何を表す数か分からない
//    if (today == 1) // 1 って何？
enum を使った場合（推奨） 
// 曜日を enum で定義
enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }

Day today = Day.MON;

// メリット：
//  - Day 型の変数には Day の値しか入れられない
//    today = 999; // コンパイルエラー
//  - 名前があるので意味が明確
//    if (today == Day.MON)
「あらかじめ決められた選択肢の中から1つだけ選ぶ」場面で enum を使うと、不正な値が入るのをコンパイル時に防げます。

■ 2. enum の宣言と使い方

● 宣言

// クラスと同じ場所に書ける
enum Day {
    MON, TUE, WED, THU, FRI, SAT, SUN
}
項目内容
キーワードenum（class と同じ位置に書く）
列挙値の書き方カンマ区切り。慣習的に全て大文字
セミコロン列挙値の最後には不要（{ } で囲うだけ）

● 代入と参照

Day today = Day.MON;       // 「列挙型名.値名」で参照
System.out.println(today); // → MON（自動的に名前が文字列化される）

// 別の値に変更
today = Day.FRI;
System.out.println(today); // → FRI
Day d = MON; のように修飾子なしで書くと、変数 MON があるのかと解釈されてコンパイルエラー。必ず Day.MON と書く。

■ 3. enum と switch（試験頻出）

enum は switch 文の条件式に使える型のひとつです。05_制御文で「switch の条件式に使える型: int, char, String, enum」と学びましたが、その enum がこれです。

Day today = Day.WED;

switch (today) {
    case MON:                   // ← Day. は不要（switch内では省略形）
        System.out.println("月曜日");
        break;
    case TUE:
        System.out.println("火曜日");
        break;
    case WED:
        System.out.println("水曜日");
        break;
    case SAT:
    case SUN:
        System.out.println("週末");
        break;
    default:
        System.out.println("その他の平日");
}
switch 内の case では Day.MON ではなく MON と書きます。Java がswitchの条件式の型から自動的に補完してくれます。

■ 4. enum の比較

enum の値は == で比較できます。同じ列挙値は同じインスタンスとして扱われるためです。

Day a = Day.MON;
Day b = Day.MON;

System.out.println(a == b);          // → true（同じインスタンス）
System.out.println(a.equals(b));     // → true（こちらでもOK）
String の == 比較は中身が同じでも false になることがあったが、enum は == で OK。文法のルールが違うので混同しないこと。

■ 実習：enum を使ってみる

実習

04_Enum フォルダに EnumDemo.java を作って実行しましょう。

// enum をクラスと同じ場所に書く
enum Signal { RED, YELLOW, GREEN }

public class EnumDemo {
    public static void main(String[] args) {
        Signal sig = Signal.RED;

        // ① 出力
        System.out.println("信号: " + sig);

        // ② switch で分岐
        switch (sig) {
            case RED:
                System.out.println("止まれ");
                break;
            case YELLOW:
                System.out.println("注意");
                break;
            case GREEN:
                System.out.println("進め");
                break;
        }

        // ③ == で比較
        if (sig == Signal.RED) {
            System.out.println("これは赤信号です");
        }
    }
}
信号: RED 止まれ これは赤信号です

● 動いたら変えてみましょう

#変えてみること確認ポイント
1Signal sig = Signal.GREEN; に変えるswitch の分岐が変わって「進め」が出力されることを確認しましょう
2Signal sig = "RED"; と書いてみるString を enum 型に代入することはできず、コンパイルエラーになることを確認しましょう
3case の中で Signal.RED と書いてみるこちらもコンパイルエラー。case の中では RED と書く必要があります

試験との対応

よく問われること正しい答え
enum はどんな型か「決まった選択肢のうちの1つ」を表す型。基本データ型ではない
switch の条件式に使えるか使える（int, char, String, enum が使える）
case 内での書き方case MON: のように修飾子なしで書く
== による比較OK（同じ列挙値は同じインスタンス）

■ 理解度チェック問題

問題04-1

曜日を表す enum Day を宣言する正しい書き方はどれか。

解説を表示 正解：イ
enum キーワードに続けて型名を書き、{ } の中に列挙値をカンマ区切りで並べる。列挙値の末尾にセミコロンは不要。クラスと同じ位置に書ける。

問題04-2

enum Day { MON, TUE, WED, THU, FRI, SAT, SUN } と宣言したとき、変数 d に「月曜日」を代入する正しい書き方はどれか。

解説を表示 正解：ウ
列挙値は「列挙型名.値名」の形で参照する。"MON" は文字列なので Day 型に代入できない。MON 単独だと変数として解釈されてエラー。enum は new でインスタンス化するものではない（生成済みの定数を選ぶイメージ）。

問題04-3

switch 文で enum を扱うとき、case ラベルの書き方として正しいものはどれか。

解説を表示 正解：エ
switch の条件式が enum 型の場合、case には列挙値の名前だけを書く。型名（Day.）を書くとエラー。Java が条件式の型から自動的に補完してくれる仕組み。

問題04-4

enum の値どうしを比較する方法として正しいものはどれか。

解説を表示 正解：ア
enum は参照型だが、各列挙値は JVM 内で1つだけ生成される（シングルトン）。Day.MON はどこから呼んでも同じインスタンスなので == で正しく比較できる。.equals() も使えるが、enum では == が慣習的に使われる（String との違いに注意）。

問題04-5

サーティファイ Java 3級の試験範囲において、enum はどのカテゴリに含まれるか。

解説を表示 正解：イ
公式試験範囲では「Javaの扱うデータ」のカテゴリに byte, short, int, long, float, double, char, boolean, enum として明記されている。本来「決まった選択肢を表す型」なので、データ型の仲間として扱われる。
