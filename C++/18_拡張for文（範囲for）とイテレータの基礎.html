16：拡張for文（範囲for）とイテレータの基礎

はじめに

この章では、C++における「拡張for文（range-based for）」の使い方を学びます。 配列やベクタのすべての要素に対して繰り返し処理を行うとき、より簡潔に記述できます。 また、後半ではイテレータの概要にも触れます。

通常のfor文との比較（16_for_vs_range.cpp）

#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> nums = {10, 20, 30, 40};

    // 通常のfor文
    for (int i = 0; i < nums.size(); i++) {
        cout << nums[i] << " ";
    }
    cout << endl;

    // 拡張for文（範囲for）
    for (int n : nums) {
        cout << n << " ";
    }
    cout << endl;
    return 0;
}

基本構文とポイント

for (データ型 変数名 : コンテナ名) {
    // 処理
}
  • 読み取り専用で使う場合は int や auto
  • 値を変更したいときは参照 int& を使う
  • 変更しない場合は const int& にすると安全・高速
for (const int& n : nums) { ... } // 読み取り専用
for (int& n : nums) { ... }       // 値を変更したい場合

範囲forで値を書き換える（16_range_modify.cpp）

#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> nums = {1, 2, 3, 4, 5};

    for (int& n : nums) {
        n *= 2;  // すべて2倍
    }

    for (int n : nums) {
        cout << n << " ";
    }
    cout << endl;
    return 0;
}

理解度チェック

問題1（16_range_const.cpp）

次のコードは何を出力しますか？

vector<int> data = {3, 6, 9};
for (const int& d : data) {
    cout << d * 10 << " ";
}
正解・解説を見る 
30 60 90
  • 値を10倍して表示しているだけで、元のベクタは変更されません。

演習問題（16_range_score_average.cpp）

5人分のテスト点数が入ったベクタ scores に対して：

  • 範囲forで全員の点数を表示しなさい
  • 範囲forで合計点を求め、平均点を表示しなさい
解答例を見る 
#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> scores = {70, 80, 90, 60, 85};

    for (int s : scores) {
        cout << s << " ";
    }
    cout << endl;

    int total = 0;
    for (int s : scores) {
        total += s;
    }
    cout << "平均: " << (double)total / scores.size() << endl;
    return 0;
}

補足：イテレータとは？（入門）

  • vector や list のようなコンテナは、要素を順にたどるために「イテレータ」という仕組みを持っています。
  • 範囲for文の内部では、自動的にイテレータが使われています。
for (auto it = vec.begin(); it != vec.end(); ++it) {
    cout << *it << " ";
}

→ *it で要素、++it で次の要素へ進む。C++のSTLでは頻繁に使われる概念です。

まとめ

  • 範囲for文を使うと、すべての要素に対する処理が簡潔に書ける
  • 値の書き換えが必要なときは参照（int&）で扱う
  • 読み取り専用なら const int& が安全
  • 範囲forは内部的にイテレータを使っている
